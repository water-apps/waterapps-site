Reference Architecture

A public summary of the delivery and runtime architecture WaterApps uses as a client-safe reference for regulated cloud programs.

Last updated: March 21, 2026 (AEDT)

Architecture Goals

  • Repeatable environment builds using Infrastructure as Code
  • Traceable CI/CD with testing and release gates before promotion
  • Strong identity, secrets, and change-control patterns
  • Operational visibility through logging, monitoring, and documented handover

Core Layers

  1. Edge and presentation layer for public entry and controlled delivery
  2. Application runtime layer for APIs, services, and isolated workloads
  3. Data and integration layer for persistence, messaging, and external systems
  4. Monitoring and audit layer for operational visibility and incident response

Delivery Controls

  • Branch protection and required CI checks
  • Automated functional and security testing
  • Identity-aware automation and least-privilege access
  • Environment-specific release controls and documentation

Typical Building Blocks

Cloud Runtime

Managed compute, controlled networking, encrypted persistence, and controlled API exposure.

Delivery Platform

Reusable workflows, policy checks, test automation, and release evidence for governed change.

Operational Layer

Monitoring, alerting, logging, runbooks, and cost visibility to support ongoing ownership.

Use This as a Conversation Starter

This is a public summary, not a customer-specific solution design. The detailed architecture and delivery scope should be shaped to your environment, risk profile, and operating model.
